fix(autophase): 3 bug-hunter findings resolved

1. CRITICAL — resume() now properly catches tick() rejection instead of
   fire-and-forget void. Handler logs to console since PhaseExecutionContext
   has no onError callback.

2. HIGH — deserializeTaskGraph now guards edges and rootNodes with ?? [],
   preventing TypeError on iteration when fields are absent in old saved data.

3. HIGH — saveToDisk: reordered read→push→write so disk is written BEFORE
   in-memory state is updated, preventing checkpoint loss if write throws.

4. MEDIUM — deleteCheckpoint is now async, awaits deleteFromDisk to avoid
   memory/disk split-brain on disk errors. Test updated accordingly.
